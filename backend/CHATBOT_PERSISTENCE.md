# Chatbot Database Integration Guide

## Overview
The chatbot now saves all conversations to the database for persistence and analytics. Messages are automatically saved when users interact with the chatbot.

---

## Database Schema

### Table: `chat_history`

```sql
CREATE TABLE IF NOT EXISTS chat_history (
    chat_id BIGINT PRIMARY KEY AUTO_INCREMENT,
    donor_id BIGINT NULL,
    user_message LONGTEXT NOT NULL,
    bot_response LONGTEXT NOT NULL,
    suggestions JSON NULL,
    session_id VARCHAR(255) NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    INDEX idx_session_id (session_id),
    INDEX idx_donor_id (donor_id),
    INDEX idx_created_at (created_at),
    FOREIGN KEY (donor_id) REFERENCES donors(donor_id) ON DELETE SET NULL
);
```

**Columns:**
- `chat_id`: Unique identifier for each chat message (Auto-increment)
- `donor_id`: Optional reference to the donor (if logged in)
- `user_message`: The message sent by the user
- `bot_response`: The response generated by the bot
- `suggestions`: JSON array of suggested quick replies
- `session_id`: Groups related messages into conversations
- `created_at`: Timestamp when the message was saved

## Backend API Endpoints

### 1. Send Message & Get Response
```
POST /api/chatbot/message
Content-Type: application/json

Request:
{
  "userMessage": "How to donate blood?"
}

Response:
{
  "userMessage": "How to donate blood?",
  "botResponse": "The blood donation process...",
  "timestamp": 1678904567000
}
```

---

### 2. Save Chat Message to Database
```
POST /api/chatbot/save
Content-Type: application/json

Request:
{
  "userMessage": "How to donate blood?",
  "botResponse": "The blood donation process...",
  "donorId": 5,                    // Optional - null if not logged in
  "sessionId": "session_1234567...",
  "suggestionsJson": "[\"Book Appointment\", \"Check Eligibility\"]"
}

Response:
{
  "status": "Chat message saved successfully",
  "chatId": 123,
  "timestamp": "2024-03-14T10:30:00"
}
```

## Frontend Implementation

### Auto-saving Messages
Messages are automatically saved to the database when the user sends them. The Chatbot component:

1. **Generates a Session ID** on first load:
   ```javascript
   // Generated format: session_{timestamp}_{random}
   // Stored in localStorage: chatbot_sessionId
   ```

2. **Detects Logged-in Donors**:
   ```javascript
   const donorId = localStorage.getItem('donorId');
   ```

3. **Saves Each Message**:
   ```javascript
   saveMessageToDatabase(userMessage, botResponse);
   ```

4. **Loads Chat History** from database on component mount

## Session Management

### Session ID Format
- **Format**: `session_{timestamp}_{randomString}`
- **Example**: `session_1710418200000_a7f8k2j1`
- **Storage**: localStorage key `chatbot_sessionId`
- **Persistence**: Lasts until browser data is cleared

### Donor Information
- **Detection**: From localStorage key `donorId`
- **Optional**: Can be null for anonymous users
- **Usage**: To track conversations per donor

## Usage Examples

### Example 1: Save User Interaction
```javascript
// Frontend sends message to backend
const response = await saveChatMessage(
  "Can I donate with diabetes?",
  "Medical conditions may affect eligibility...",
  null,  // Anonymous user
  "session_1234567abc",
  ["Check Eligibility", "Contact Support"]

// Message is saved to database with timestamp
console.log(`Chat saved with ID: ${response.data.chatId}`);
```

```javascript
  { text: chat.userMessage, sender: "user" },
  { text: chat.botResponse, sender: "bot" }
```

```javascript
// Admin can view all conversations for a specific donor
```

## Performance Considerations

### Database Indexes
- `idx_session_id`: Fast retrieval of messages by session
- `idx_donor_id`: Fast retrieval of donor conversations
- `idx_created_at`: Fast sorting and time-range queries

### Recommended Cleanup (Optional)
For production, consider archiving old chat history:

```sql
-- Archive messages older than 90 days
INSERT INTO chat_history_archive 
SELECT * FROM chat_history 
WHERE created_at < DATE_SUB(NOW(), INTERVAL 90 DAY);

-- Delete archived messages
DELETE FROM chat_history 
WHERE created_at < DATE_SUB(NOW(), INTERVAL 90 DAY);
```

## Troubleshooting

### Chat History Not Saving
1. Verify backend is running on port 8080
2. Check browser console for errors
3. Ensure sessionId is generated: `localStorage.getItem('chatbot_sessionId')`
4. Verify database table exists: `DESCRIBE chat_history;`

### Can't Load Chat History
1. Check if sessionId exists in localStorage
2. Verify the sessionId was used to save messages
3. Check database has records: `SELECT COUNT(*) FROM chat_history WHERE session_id = '{sessionId}';`

### Database Connection Error
1. Verify MySQL is running
2. Check `application.properties` database configuration
3. Ensure `blood_donation_system` database exists
